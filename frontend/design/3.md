## Brand & Style

The design system is engineered for the modern SME, bridging the gap between high-powered enterprise functionality and a calm, approachable user experience. The brand personality is **reliable, precise, and sophisticated**, designed to reduce the cognitive load inherent in ERP workflows. 

The aesthetic is a refined evolution of corporate minimalism. It utilizes a **Modern Enterprise** style—characterised by generous white space, a structured information hierarchy, and a "soft-professional" tactile quality. The goal is to evoke a sense of total control and clarity, transforming complex data entry and resource planning into a streamlined, high-end digital experience.

## Colors

The palette is anchored by **Indigo-600 (#4F46E5)** as the primary driver for action and focus. This is contrasted against a sophisticated scale of slates. 

- **Primary Indigo:** Used for primary actions, active states, and brand signatures.
- **Surface & Canvas:** The application uses a "Layered White" approach. The main canvas is **Slate-50**, while functional cards and containers are pure **White** to create a subtle natural elevation.
- **Semantic Logic:** Success, Warning, and Error colors are used sparingly for status badges, progress bars, and destructive modals to ensure high "signal-to-noise" ratio.
- **Typography Colors:** Use **Slate-900** for headings to ensure maximum contrast, and **Slate-600** for secondary body text.

## Typography

The design system utilizes **Inter** exclusively to leverage its exceptional legibility in data-heavy environments. 

The type scale is optimized for **high information density**. 14px is the standard body size for general text, while 13px is utilized for data tables and sidebars to maximize vertical real estate. Bold weights (600/700) are reserved for structural headings and primary identifiers. Use uppercase labels for small metadata or section headers to provide clear visual separation without increasing font size.

## Layout & Spacing

The layout is a **Desktop-First Fixed Grid** logic intended for high-productivity dashboard environments.

- **Main Structure:** A collapsible left-hand navigation sidebar (260px expanded) paired with a fixed top utility header (64px).
- **Grid:** Content is housed within a fluid container with a max-width of 1600px, utilizing a 12-column system. Gutters are fixed at 16px to maintain density while preventing visual clutter.
- **Density:** Spacing follows a 4px baseline. Components like data tables should use compact padding (8px - 12px) to ensure users can see as many rows as possible without scrolling.
- **Breakpoints:**
  - **Desktop (1280px+):** Full sidebar, 12-column grid.
  - **Tablet (768px - 1279px):** Collapsed sidebar (icons only), 8-column grid.
  - **Mobile (<767px):** Hidden sidebar (hamburger menu), 4-column grid, stacked data cards instead of tables.

## Elevation & Depth

The design system uses **Tonal Layering** combined with **Ambient Shadows** to create a professional, non-distracting hierarchy.

- **Level 0 (Canvas):** Slate-50 (#F8FAFC). Background for the entire application.
- **Level 1 (Surface):** White (#FFFFFF). Standard background for cards, tables, and main content blocks. Uses a 1px border of Slate-200.
- **Level 2 (Popovers):** White. Used for dropdowns, tooltips, and fly-outs. Features a soft, diffused shadow: `0 4px 12px -2px rgba(15, 23, 42, 0.08)`.
- **Level 3 (Modals):** White. Highest elevation. Features a heavy shadow: `0 20px 25px -5px rgba(15, 23, 42, 0.1)`.

Avoid heavy blurs or vibrant glassmorphism to maintain focus on data integrity.

## Shapes

The shape language is **Structured and Professional**. 

A standard border radius of **8px (0.5rem)** is used for all primary containers, input fields, and buttons. This provides a modern, "soft-enterprise" feel that is more approachable than sharp corners but more serious than pill-shaped designs. Large layout cards or modal containers may use **16px (1rem)** for a more premium, framed appearance.

## Components

- **Data Tables:** The core of the ERP. Features sticky headers, hover-state highlights in Slate-50, and 1px horizontal dividers in Slate-200. Cell padding is tight (12px horizontal, 8px vertical).
- **Buttons:** 
  - *Primary:* Solid Indigo-600 with white text. 
  - *Secondary:* White background, Slate-200 border, Slate-700 text.
  - *Tertiary:* Ghost style, no border, Indigo text for "Save/Edit" or Slate text for "Cancel."
- **Input Fields:** 1px Slate-300 border, 8px roundedness. Active state uses a 2px Indigo-600 focus ring with a 2px offset.
- **Status Badges:** Subtle backgrounds (e.g., Emerald-50) with high-contrast text (Emerald-700). Pill-shaped for distinctiveness from rectangular buttons.
- **Sidebar Nav:** High-contrast Slate-900 background or very light Slate-50 depending on the specific module. Active links use a 4px vertical Indigo stripe on the left edge.
- **Cards:** Used for grouping related data (e.g., "Customer Info," "Order Totals"). Always white with a 1px Slate-200 border.
